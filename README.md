# Open-AI.ru - Портал для автоматизации МСП

## Обзор проекта

Open-AI.ru - это платформа, объединяющая стартапы с открытым ИИ и специалистов, которые помогают автоматизировать производственные и организационные процессы в малых и средних предприятиях (МСП) с использованием SaaS-решений и ИИ-инструментов.

## Цели проекта

- Создать экосистему для связи разработчиков ИИ-решений с предприятиями
- Ускорить цифровизацию МСП через готовые решения
- Стимулировать развитие отечественных ИИ-технологий
- Создать площадку для обмена опытом и знаниями

## Архитектура решения

### Основные компоненты
- **Веб-платформа** - основной интерфейс взаимодействия
- **API** - программные интерфейсы для интеграции
- **AI Agent Sandbox** - безопасная среда для тестирования ИИ-агентов
- **База знаний** - образовательный контент и документация
- **Система аналитики** - метрики и рекомендации

### Технологический стек
- **Frontend**: React/Vue.js, TypeScript
- **Backend**: Node.js/Python, микросервисная архитектура
- **База данных**: PostgreSQL/MongoDB
- **ИИ-интеграции**: OpenAI, YandexGPT, отечественные решения
- **Инфраструктура**: AWS/Azure, Docker, Kubernetes

## Ролевая модель

### Стартапы
Компании, предлагающие ИИ-решения и SaaS-продукты для автоматизации бизнес-процессов.

### Специалисты
Индивидуальные эксперты в области ИИ, автоматизации и цифровой трансформации.

## AI Agent Sandbox

Безопасная изолированная среда для запуска и тестирования ИИ-агентов. Полная документация находится в [sandbox/README.md](sandbox/README.md).

### Особенности
- Docker-based контейнеризация с полной изоляцией
- Ограничения ресурсов (CPU, RAM, timeout)
- Rate limiting и защита от DoS атак
- REST API для управления сессиями
- React веб-интерфейс
- Демонстрационный агент анализа текста

### Быстрый старт

```bash
cd sandbox/docker
docker-compose up
```

- Frontend: http://localhost:3000
- Backend API: http://localhost:3001

См. [sandbox/README.md](sandbox/README.md) для детальной документации.

## Требования к запуску

### Системные требования
- Node.js 18+
- Docker и Docker Compose
- PostgreSQL 13+
- Redis (для кэширования)

### Установка и запуск
```bash
# Клонирование репозитория
git clone https://github.com/G-Ivan-A/open-ai.ru.git
cd open-ai.ru

# Установка зависимостей
npm install

# Запуск в режиме разработки
npm run dev

# Сборка для продакшена
npm run build
```

## Разработка

### Соглашения по коду
- Использование TypeScript для типизации
- Следование принципам SOLID
- Комментарии на русском языке

3. Code review и тестирование

## Тестирование

### Виды тестирования
- **Unit-тесты** - тестирование отдельных компонентов
- **Integration-тесты** - тестирование взаимодействия сервисов
- **E2E-тесты** - полное тестирование пользовательских сценариев
- **Performance-тесты** - нагрузочное тестирование

### Запуск тестов
```bash
# Unit-тесты
npm run test:unit

# Integration-тесты
npm run test:integration

# E2E-тесты
npm run test:e2e

# Все тесты
npm test
```
