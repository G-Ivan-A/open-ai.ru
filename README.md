# Open-AI.ru - Открытая платформа для open-source решений

## Обзор проекта

**open-ai.ru** — открытая платформа для open-source решений по отраслям.

Платформа объединяет разработчиков open-source решений и малый/средний бизнес (МСП), предоставляя готовые ИИ-агенты и инструменты для автоматизации бизнес-процессов. На старте проекта — раздел ИИ-агенты для создания порталов малого бизнеса + раздел репутационных технологий как часть экосистемы.

## Ключевые принципы

- **Оценка инструментов, а не участников** — фокус на практической пользе решений
- **Прозрачность ИИ-влияния** — пользователь понимает, как работают алгоритмы
- **Open-source приоритет** — открытые решения с возможностью монетизации через допуслуги
- **Архитектура независимости** — отказоустойчивость компонентов
- **Защита от спама** — обсуждение только через ИИ-агентов в песочнице

## Технологический стек

### Frontend
- **Framework**: Next.js 14+ (React) с TypeScript
- **State Management**: Zustand + React Query
- **UI**: Tailwind CSS + shadcn/ui
- **Deployment**: Vercel

### Backend
- **Runtime**: Node.js 20 LTS
- **Framework**: NestJS
- **API**: REST + GraphQL (гибридный подход)
- **Deployment**: AWS / Yandex Cloud

### База данных
- **Основная БД**: PostgreSQL 15+ с Prisma ORM
- **Кэширование**: Redis 7+
- **Поиск**: PostgreSQL Full-Text (MVP) → Elasticsearch (позже)

### Инфраструктура
- **Контейнеризация**: Docker + Docker Compose
- **CI/CD**: GitHub Actions
- **Мониторинг**: Sentry, Prometheus + Grafana

Подробная архитектура описана в [ARCHITECTURE.md](ARCHITECTURE.md).

## Быстрый старт

### Системные требования
- Node.js 20 LTS
- Docker и Docker Compose
- Git

### Установка и запуск

```bash
# Клонирование репозитория
git clone https://github.com/G-Ivan-A/open-ai.ru.git
cd open-ai.ru

# Установка зависимостей для frontend
cd apps/frontend
npm install

# Установка зависимостей для backend
cd ../backend
npm install

# Запуск всех сервисов через Docker Compose
cd ../..
docker-compose up -d

# Или запуск локально
# Frontend (в одном терминале)
cd apps/frontend
npm run dev

# Backend (в другом терминале)
cd apps/backend
npm run start:dev
```

Frontend будет доступен на http://localhost:3000
Backend API на http://localhost:4000

### Переменные окружения

Создайте `.env` файлы в соответствующих директориях:

**apps/frontend/.env.local**:
```env
NEXT_PUBLIC_API_URL=http://localhost:4000
NEXTAUTH_SECRET=your-secret-key
NEXTAUTH_URL=http://localhost:3000
```

**apps/backend/.env**:
```env
DATABASE_URL=postgresql://user:password@localhost:5432/openai_ru
REDIS_URL=redis://localhost:6379
JWT_SECRET=your-jwt-secret
```

### Соглашения по коду

- **Язык**: TypeScript в strict mode
- **Стиль**: ESLint + Prettier
- **Commits**: Conventional Commits
- **Комментарии**: На русском для бизнес-логики
- **Тестирование**: Минимум 80% покрытие

### Процесс разработки

1. Создайте issue для новой функциональности
2. Создайте feature-ветку: `git checkout -b feature/название-фичи`
3. Разработка с регулярными коммитами
4. Запустите тесты: `npm test`
5. Создайте Pull Request
6. Code review минимум от 1 человека
7. Merge в main после прохождения CI/CD

Подробнее см. [CONTRIBUTING.md](CONTRIBUTING.md)

## Тестирование

### Виды тестирования
- **Unit-тесты** - тестирование отдельных компонентов
- **Integration-тесты** - тестирование взаимодействия сервисов
- **E2E-тесты** - полное тестирование пользовательских сценариев
- **Performance-тесты** - нагрузочное тестирование

### Запуск тестов
```bash
# Unit-тесты
npm run test:unit

# Integration-тесты
npm run test:integration

# E2E-тесты
npm run test:e2e

# Все тесты
npm test
```
