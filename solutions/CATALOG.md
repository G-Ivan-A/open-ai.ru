# Каталог open-source ИИ-решений для МСП

Репозиторий **open-ai.ru** — не маркетплейс, а куратор и оркестратор. Мы собираем, тестируем и оркестрируем лучшие open-source ИИ-решения, делая их доступными для развёртывания от ноутбука сотрудника до облачного кластера.

---

## Уровни готовности к развёртыванию

| Уровень | Среда | Требования |
|---------|-------|------------|
| **Starter** | Ноутбук / рабочий компьютер | 8 ГБ RAM, CPU, без GPU |
| **Business** | Корпоративный сервер | 32+ ГБ RAM, опционально GPU |
| **Cloud Pro** | Среднее облако | 4–8 vCPU, 16–64 ГБ RAM, GPU-инстанс |
| **Cloud Enterprise** | Мощное облако | Кластер, GPU-ноды, MLOps-пайплайн |

---

## Критерии отбора проектов

Проект попадает в каталог при соответствии **всем обязательным** критериям и хотя бы **двум дополнительным**.

### Обязательные критерии

1. **Открытая лицензия** — MIT, Apache 2.0, GPL, AGPL или иная OSI-одобренная лицензия.
2. **Активная разработка** — коммит в репозиторий не старше 6 месяцев.
3. **Применимость для МСП** — решает реальную бизнес-задачу без требования специализированной ML-команды.
4. **Документация** — README с инструкцией по запуску, описание API или UI.

### Дополнительные критерии

- Поддержка Docker / Docker Compose или Helm-чарт.
- Наличие русскоязычного интерфейса или возможность локализации.
- Совместимость с локальными LLM (Ollama, LM Studio, локальный Llama).
- Интеграция с популярными российскими или международными инструментами (1С, amoCRM, Bitrix24, Notion, Slack).
- Наличие demo-режима или sandbox без регистрации.
- Присутствие в топ-1000 репозиториев GitHub по тематике ИИ.

### Исключающие критерии (автоматический отказ)

- Проект заброшен (нет активности более 12 месяцев).
- Проприетарная лицензия на ключевые компоненты.
- Требует постоянного доступа к платному внешнему API без локальной альтернативы.
- Отсутствие документации по установке.

## Шаблон карточки проекта

Каждый проект описывается в отдельной директории `solutions/<project-slug>/`. Структура:

```
solutions/<project-slug>/
├── README.md          — карточка проекта (по шаблону)
├── deployment/
│   ├── docker-compose.yml  — готовый конфиг для быстрого старта
│   └── .env.example         — переменные окружения
└── limitations.md     — известные ограничения и workarounds
```

Шаблон `README.md` — см. файл [SOLUTION_TEMPLATE.md](SOLUTION_TEMPLATE.md).

## Оркестрационный слой

Репозиторий включает набор **Deployment Templates** — готовых конфигураций для совместного запуска нескольких решений в мульти-агентных сценариях:

| Шаблон | Состав | Сценарий |
|--------|--------|----------|
| `starter-rag-stack` | Open WebUI + Ollama | Локальный корпоративный чат с документами |
| `business-rag-stack` | AnythingLLM + Clarify Engine + n8n | Автоматизация обработки запросов |
| `cloud-enterprise-stack` | RAGFlow + LlamaIndex + Langfuse | Продвинутая RAG-система с мониторингом |

Шаблоны находятся в директории `deployment/`.

## Процесс добавления нового решения

1. Открыть Issue с меткой `catalog: propose` и заполнить чек-лист критериев.
2. После обсуждения — создать директорию `solutions/<slug>/` по шаблону.
3. Проект проходит тестирование силами maintainer'ов или контрибьюторов.
4. После валидации карточка публикуется в реестре.
