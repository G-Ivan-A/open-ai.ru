# Clarify Engine — Известные ограничения

## Производительность

### Без GPU (Starter)
- Время ответа: 10–60 секунд в зависимости от модели и длины контекста.
- Рекомендуемые модели для CPU: `llama3.2:3b`, `phi3:mini`, `mistral:7b` с Q4-квантизацией.
- Параллельные запросы замедляют работу — оптимально для 1–3 одновременных пользователей.

**Workaround**: использовать более лёгкие модели (3B параметров вместо 7B) или перейти на конфигурацию Business с GPU.

### Большие документы
- Документы более 500 страниц могут требовать увеличения таймаута при индексации.
- PDF со сложной вёрсткой (таблицы, колонки) могут парситься с потерей структуры.

**Workaround**: разбивать документы на логические части перед загрузкой; для сложных PDF использовать предварительную конвертацию через `pdf2docx`.

## Форматы и парсинг

- **Сканированные PDF** (без текстового слоя): не поддерживаются на Starter. Требуют OCR (Tesseract или коммерческое решение) — доступно на уровне Business.
- **Excel / PPTX**: частичная поддержка, таблицы конвертируются в текст с потерей структуры.
- **Кириллица в именах файлов**: в некоторых конфигурациях Windows вызывает ошибки парсинга. Workaround: использовать латиницу в именах файлов.

## Точность ответов

- Качество ответов напрямую зависит от выбранной LLM: более крупные модели дают лучший результат.
- При плохом качестве текста в документе (сканы, некорректное форматирование) точность снижается.
- Вопросы, требующие сравнения данных из разных документов, могут давать неполные ответы — RAG-архитектура оптимизирована для извлечения, а не анализа.

## Масштабирование

- ChromaDB (встроенная база) не рекомендуется при объёме более 100 000 документных чанков — переходить на Qdrant.
- Конфигурация Starter не рассчитана на более 5 одновременных пользователей.

## Безопасность

- Аутентификация в базовой конфигурации — простой токен. Для корпоративного использования требуется настройка SSO или обратного прокси (nginx + basic auth).
- Доступ к API не ограничен по умолчанию — рекомендуется настроить firewall при публичном развёртывании.
