# Состав команды, роли и иерархия решений: open-ai.ru

## 1. Фаундер (Иван Гулиенко) — расширенная роль

Полноценный Product Owner + Domain Expert + Final Approver.

| Аспект | Роль |
|--------|------|
| **Стратегия** | Видение, moat, долгосрочная цель |
| **Продуктовые решения** | Что делаем, что не делаем, приоритеты |
| **Бизнес-логика** | Ценность, целевые аудитории, модель монетизации |
| **Дословные правки** | Якоря формулировок, от которых не отступаем |
| **Утверждение** | PR review, merge, финальная валидация |
| **Ресурсы** | Кто когда работает, приоритезация команд |
| **Коммуникация** | Диалог с внешними командами (C, G, D, O) |
| **Экспертиза** | Домен БА ИТ/телеком, методология hybrid intelligence |
| **Прагматизм** | Баланс между идеалами и реальностью, защита от over-engineering |
| **Окончательные решения** | При противоречиях побеждает мнение фаундера |

## 2. AI-команды (5 команд)

Представлены как мини-фирмы экспертов.

| Код | Название | Модель | Роль | Расширенное описание (экспертиза) |
|-----|----------|--------|------|-----------------------------------|
| **C** | Прорыв | OpenAI (ChatGPT/Codex) | Стратегические инсайты | Команда инноваторов: генерирует радикально новые подходы, синтезирует междисциплинарные инсайты, предлагает "что если" сценарии. Фокус: originality + feasibility. |
| **Q** | Поддержка | Qwen | Консультант, стратег | Команда консультантов: строит чёткие фреймворки, обеспечивает coherence, переводит идеи в actionable plans, структурная опора. |
| **G** | Исследователь | Grok | Исследования, проверка гипотез | Команда исследователей: глубокие поиски, fact-checking, тестирование гипотез на данных, выявление blind spots. |
| **D** | Аудит | DeepSeek | Перекрёстная валидация | Команда аудиторов: независимая cross-check всех outputs, поиск уязвимостей, stress-testing, cost-benefit analysis. |
| **O** | Мыслитель | Claude Opus | Глубокий анализ | Команда глубоких мыслителей: многошаговый анализ, разбор trade-offs, ethical implications, holistic understanding сложных систем. |

## 3. Бот-исполнитель

- **@konard** — работает с командами C или O. Исполнитель в репо (PR, код, документы). Не принимает стратегических решений, реализует задачи.

## 4. Иерархия решений

```text
Фаундер > Команда Q > @konard > Команда C > Команда O > Команда G > Команда D
```

*При противоречиях — решение фаундера окончательное.*

## 5. Workflow (Как работает команда)

1. Фаундер задаёт направление и якоря.
2. Команда C генерирует прорывные идеи.
3. Команда Q структурирует идеи в рамки и планы.
4. Команды G и O проводят исследование и глубокий анализ.
5. Команда D проводит аудит и ищет ошибки.
6. @konard реализует согласованное в репозитории.
7. Фаундер утверждает результат.
