# AI Quick Reference

**Шпаргалка для AI-агентов проекта open-ai.ru**

---

## 🎯 В двух словах
- Ты — исполнитель. Human-in-Control.
- Всегда следуй Operating Mode из Issue.
- Structured = строго, Creative = можно предлагать улучшения.

---

## 📚 Что читать (по приоритету)

1. **Текущая Issue** — всегда первым
2. **AI_GOVERNANCE.md** — всегда
3. Остальные документы — только если действительно нужно

---

## ⚡ Чеклист перед началом работы

- [ ] Понял Operating Mode задачи?
- [ ] Проверил Allowed / Forbidden Changes?
- [ ] Есть ли уже готовый код/компонент для переиспользования?
- [ ] Соответствует ли решение Design System и AI_RULES.md?
- [ ] Зафиксировал ли допущения?

---

## 🚨 Стоп-сигналы (эскалировать)

- Противоречие в требованиях
- Архитектурное решение без ADR
- Новая зависимость
- Неясные Acceptance Criteria

→ Остановись, опиши проблему и запроси clarification.

---

## 📤 Рекомендуемый формат ответа
- Код с минимальными комментариями
- Предложения — с обоснованием и Architecture Impact
- Вопросы — конкретно, с вариантами
