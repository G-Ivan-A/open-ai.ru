# Repository Labels

This document proposes the baseline label set for repository governance.

## Foundation Labels

| Label | Purpose | Suggested color |
| --- | --- | --- |
| `foundation` | Repository structure, governance, and AI-native SDLC foundations. | `0e8a16` |
| `governance` | Review policies, templates, process, and decision traceability. | `5319e7` |
| `repository-setup` | Bootstrapping, cleanup, CI, and repository configuration. | `1d76db` |
| `ai-native` | AI-assisted workflows, agent contracts, prompts, and reviews. | `bfdadc` |
| `cleanup-required` | Work that must remove or isolate legacy artifacts first. | `d93f0b` |
| `high-priority` | Work requiring expedited founder or maintainer attention. | `b60205` |
| `needs-human-review` | Changes that require explicit human review before merge. | `fbca04` |

## Product Labels

| Label | Purpose | Suggested color |
| --- | --- | --- |
| `feature` | User-facing product capability. | `1d76db` |
| `research` | Investigation or decision support before implementation. | `c5def5` |
| `contracts` | API, event, or AI-agent contract changes. | `0052cc` |
| `security` | Security, privacy, access, or threat model work. | `ee0701` |
| `documentation` | Documentation-only changes. | `0075ca` |
